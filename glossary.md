
# Glossary

**Action**: atomic, immediately doable step
**Routine**: recurring definition that generates actions
**Action Template**: description of an action inside a routine
**Task**: finite obligation, fulfilled via actions
**Project**: container for tasks
**Trigger**: condition for routine execution
**Cadence**: recurrence pattern
**Postponed**: action deferred
**Rejected**: action skipped, not a failure
**Engagement**: user interaction with the system
